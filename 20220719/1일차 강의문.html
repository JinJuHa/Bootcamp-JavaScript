변수 이름을 통해 값의 의미를 명확히 알 수 있어 가독성이 좋아진다 1-1. 선언 방법 - var 키워드 사용하여 변수 선언(읭?) var x; //변수의 선언 1-2. 할당 방법 - 할당 연산자 = 를 사용하여 할당 x = 8; //할당 2. 값 var str = 'Hello Meta'; - str 이라는 이름의 변수를 선언하고 - 문자열 리터럴 'Hello Meta' 를 값으로 할당하였다. 값은 프로그램에 의해 조작될 수 있는 대상을 뜻한다 데이터 타입 : 프로그래밍 언어에서 사용할 수 있는 값의 종류 변수 : 값이 저장된 메모리 주소를 가리키는 식별자 리터럴 : 소스코드 안에서 직접 만들어 낸 상수 값 자체를 말하며 값을 구성하는 최소 단위 3. 연산자 - 연산의 대상을 피연산자라 함 4. 주석(comment) - 작성된 코드의 의미를 설명하기 위해 사용 5. 표현식 - 하나의 값으로 평가 - 복잡하게 구성 가능 6. 문 - 단계별로 수행될 명령들의 집합. ;로 끝나야 한다 - 조건문, 반목문으로 제어함 *문과 표현식 비교 - 표현식을 통해 평가한 값을 컴튜터에게 명령을 하여 무언가를 하는 것은 문이다 7. 함수 - 어떤 작업을 수행하기 위해 필요한 문들의 집합을 정의한 코드 블록이다 - 이름과 매개변수. 블록에 담긴 문들을 일괄적으로 실행 8. 객체 - 자바스크립트는 객체 기반 언어 -키(이름)와 값으로 구성된 프로미터의 집합 9. 배열(Array) - 1개의 변수에 여러개의 값을 순차적으로 저장할 때 사용 ---------------------------------------------------------------------------------- 데이터 타입 & 변수 1. 데이터 타입 -메모리 저장을 위해 공간을 확보할 메모리 크기를 알아야 하고 값의 종류에 따라 확보할 메모리 크기가 다르다 이때의 값의 종류를 데이터 타입이라고 한다 -메모리에 저장되어 있는 2진수 데이터를 어떻게 해석할지에 대한 정보를 제공 1-1. 원시타입 - 변경 불가능한 값이며 값에 의한 전달이다 1-1-1. 숫자 하나의 숫자 타입만 존재 모든 수를 실수로 처리 십진수로만 표기됨 1-1-2. 문자열 - 텍스트를 나타내는데 사용한다 - 일반적으로 작은 따옴표를 사용한다 1-1-3. 불리언 -논리문의 참과 거짓을 나타낸다 (true, false) 1-1-4. undefined -선언 이후 값이 할당하지 않은 변수는 언디파인드 값을 가진다 1-1-4. null -대소문자로 구분하므로 NULL이랑은 다르다 -없다는 값을 표시하는 것으로 언디파인드와는 다르다 1-1-6. 심볼 -ES6에서 추가된 원시타입 1-2. 객체타입 - 참조의 의한 전달 방식으로 전달된다 - 객체는 데이터와 그 데이터에 관련된 동작 모두 포함할 수 있는 개념적 존재 - 이름과 값을 가지는 데이터를 의미하는 프로퍼티와 동작을 의미하는 메소드를 포함할 수 있는 독립적 주체 - 주소를 공유하여 하나를 바꾸면 함께 바뀐다 2. 변수 - 데이터를 기억하여 필요한 때에 재사용하기 위해 고유의 이름인 식별자를 명시한 것이다 - 변수는 var, let, const 키워드를 사용하여 선언 - 할당 연산자를 사용해 값을 할당한다 - 식별자인 변수명을 사용해 변수에 저장된 값을 참조한다 2-1. 변수의 선언 - 한번 쓰고 버리는 값이 아닌 일정기간 유지할 필요가 있는 경우에만 변수 사용 선언하지 않는 변수에 접근하면 ReferenceError (로그로 찍지 않은 값) 2-2. 변수의 중복선언 - var는 중복 선언이 가능함 - 그러나 에러가 발생할 경우를 생각하여 중복선언을 하지 않는 것이 좋다 2-3. 동적 타이핑 - 자바스크립트는 동적 타이핑 언어이다 - 값이 할당되는 과정에서 값의 타입에 의해 자동으로 타입이 결정된다 2-4. 변수 호이스팅 - var 선언문이나 function 선언문 등 모든 선언문이 Scope의 선두로 옮겨져 동작하는 특성 - 모든 선언문이 선언되기 이전에 참조 가능하다 선언단계 > 초기화 단계 > 할당 단계 2-4. var 키워드로 선언된 변수의 문제점 - 함수 레벨 스코브 - 전역 변수의 남발 - var 키워드 생략 허용 - 의도하지 않은 변수의 전역화 - 중복 선언 허용 - 의도하지 않은 변수값 변경 - 변수 호이스팅 - 변수를 선언하기 전에 참조가 가능하다 여러 문제가 있어 지금은 let, const 사용 -------------------------------------------------------------------------------------------- 1. 연산자 - 하나의 값을 만든다. 1-1. 산술 연산자 - 수학 계산을 통해 새로운 숫자 값을 만든다 - 산술 연산을 할 수 없을 땐 NaN을 반환한다 1-1-1. 이항 산술 연산자 - 2개의 피연산자를 산술 연산하여 숫자 값을 만든다 1-1-2. 단항 산술 연산자 - 1개의 피연산자를 대상으로 연산 - 증가/감소 연산자는 위치에 따라 피연산자의 값을 변경하는 부수 효과가 있다 피연산자 앞에 위치한 전위증가/감소 연산자는 피연산자의 값을 증가/감소 시킨 후 다른 연산 수행 피연산자 뒤에 위치한 후위증가/감소 연산자는 먼저 다른 연산을 수행한 후, 피연산자의 값을 증가/감소 시킨다 1-1-3. 문자열 연결 연산자 -연산자는 피 연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작 1-2. 할당 연산자 1-3. 비교연산자 - if, for 문과 같은 제어문의 조건식에서 사용한다 1-3. 동등 / 일치 비교 연산자 - 동등비교 연산자는 피연산자를 비교할 때 타입을 일치시킨 후 비교한다 일치비교(==)연산자는 피연산자가 타입도 같고 값도 같은 경우에만 true를 반환 1-3-1. 대소관계 비교연산자 >= 1.4. 삼항 조건 연산자 - 조건식의 평가 결과에 따라 반환할 값을 결정한다 var result = score >= 60 ? 'pass' : 'false'; score가 60보다 크고 작은지에따라 출력되는 글자가 pass,false가 나옴 1-5. 논리 연산자 - 피연산자(부정 논리의 연산자인 경우, 우항의 피연산자)를 논리 연산한다 || 논리합 연산자 && 논리곱 ! 부정 1-6. 쉼표 연산자 - 왼쪽 피연산자부터 차례로 피연산자를 평가 1-8. typeof 연산자 - 자신 뒤에 위치한 피연산자의 데이터 타입을 문자열로 반환한다 (string, number 등등 문자열) -------------------------------------------------------------------------------------------- < 과제 > 1. 공통 - 교재 읽어오기 : 제어문(93 ~ 118) 객체(124 ~ 153) 1-1.차 - 예제 타이핑 : 예제07-01 ~ 예제 07-33 --------------------------------------------------------------------------------------------- 제어문 1. 제어문 - 주어진 조건에 따라 코드 블록을 실행(조건문)하거나 반복실행(반복문)할 때 사용한다 - 일반적인 코드는 위아래로 이동함 1-1. 블록문 - 0개 이상의 문들을 중괄호로 묶는 것으로 블록문을 하나의 단위로 취급한다 - 문의 끝에 세미콜론;을 붙이는 것이 일반적이지만 블록문은 붙이지 않는다 1-2. 조건문 - 주어진 조건식의 평가 결과에 따라 코드블럭(블록문)의 실행을 결정한다 - 조건식은 블리언 값으로 평가될 수 없는 표현식이다 1-2-1. if, ...else 문 - 주어진 조건식의 평가 결과에 따라 실행할 코드 불록을 결정 - 조건식의 평가 결과가 불리언 값이 아니라면 불리언 값으로 강제 변환되어 구별 - 간단하면 중괄호 생략 가능
